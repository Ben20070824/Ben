AIChat
# 一、项目概述
本项目是一款仿写豆包的AIChat应用，核心功能与基础功能已全部实现，加分功能完成部分开发，自己额外加了一个账号管理（参照寒假的项目，不过这次用数据库实现账号管理），API对接的是deepseek
# 二、核心功能与技术实现
## （一）主要功能
1. 账号管理：通过Room数据库持久化存储用户信息，实现账号相关的基础操作，保证每个账号的数据独立性。
2. AI对话：基于Retrofit+Rxjava实现网络请求，支持上下文关联，对话内容可持久化存储，确保聊天连贯性。
3. 智能体：支持角色设定与自定义prompt，智能体数据可存储，且与账号绑定。
## （二）核心技术与整体逻辑
1.  数据存储：采用3个Room数据库分别管理不同类型数据（账号、对话、智能体），来实现存储。
2.  页面交互：通过RecyclerView的点击事件传参加载对应布局，页面跳转时通过参数传递，实现布局动态加载与数据联动。
# 三、项目亮点（仅代表个人意见）
- 智能体设定：支持自定义角色和prompt，满足用户个性化对话需求；
- 聊天气泡设计： 用9-patch实现

# 六、项目心得体会
本次项目开发让我对Android相关技术有了更深入的实践，同时也暴露了自身的诸多问题：
- 知识记忆与运用不足：对Android大部分知识点能理解，但难以熟练记忆（除源码外，源码理解难度较高），导致开发效率偏低；例如Room数据库，写了三次，才逐渐得心应手。
- 知识理解不透彻：部分代码仅能“照猫画虎”，未能深入理解原理，比如ViewModel初始化的委托方式by viewModels()，目前仅会使用，尚未理解其底层逻辑。
- 业务逻辑把控不足：开发过程中偶尔出现逻辑混乱、顾此失彼的情况，核心原因在于demo写得不够，对业务场景的梳理能力有待提升。
- 项目效率有待提升：本次项目耗时较长，更多是依靠时间堆砌完成，若能熟练运用相关技术，可提升开发效率，实现更多功能。
# 七、选题理由
选择开发AIChat应用，主要有两个核心原因：一是自身网络请求相关知识掌握不够扎实，希望通过实际项目锻炼Retrofit、Rxjava等网络请求技术的运用；二是当前AI风口盛行，AIChat场景贴近实际需求，开发过程中既能巩固技术，也能接触到贴合行业趋势的应用场景。
